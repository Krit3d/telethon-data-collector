# Telethon Channel Data Collector (TCDC)

Асинхронный парсер Telegram-каналов на базе Telethon и PostgreSQL (SQLAlchemy 2.0). 
Разработан как базовый модуль для системы семантического анализа. Реализована обработка сетевых ошибок, обход `FloodWaitError` и паттерн Upsert для базы данных.

## 🚀 Основные возможности
- **Asynchronous Engine:** Использование `Telethon` и `SQLAlchemy 2.0 (asyncio)` обеспечивает максимальную производительность.
- **Resilience & Fault Tolerance:** 
  - Автоматическая обработка `FloodWaitError` (умное ожидание лимитов).
  - Экспоненциальная задержка при сетевых ошибках (retries).
- **Data Integrity:** Реализована логика **Upsert (On Conflict Do Update)** — данные о постах и каналах (просмотры, реакции) обновляются без создания дубликатов.
- **Concurrency Control:** Использование `asyncio.Semaphore` для предотвращения блокировок аккаунта при массовом парсинге.
- **Dockerized:** Полная изоляция окружения и быстрый запуск через Docker Compose.

### Ограничения
В текущей реализации скрипт настроен на работу с демонстрационным списком каналов. Парсинг 1000+ каналов с одного личного аккаунта в сжатые сроки не производился намеренно, чтобы избежать перманентной блокировки (Flood Ban) со стороны Telegram API. Для расширения функционала архитектура предусматривает использование пула аккаунтов и ротацию прокси-серверов.

## 🛠 Технологии
- **Python 3.12+**
- **Telethon** (Асинхронное взаимодействие с Telegram API)
- **PostgreSQL + asyncpg** (Асинхронный драйвер БД)
- **SQLAlchemy 2.0** (ORM)
- **Docker & Docker Compose**

## 🏗 Архитектурные решения
- **Модульность:** Логика БД полностью отделена от логики парсера, что позволяет легко заменить PostgreSQL на векторную БД (например, Pinecone или ChromaDB) для задач ИИ.
- **Масштабируемость:** Структура таблиц `channels` и `posts` поддерживает хранение миллионов записей и готова к индексации для полнотекстового или семантического поиска.
- **Логирование:** Использован стандартный модуль `logging` с разделением уровней (INFO/ERROR), что упрощает мониторинг в контейнере.
- **Unique constraint**: При построении БД был использован составной уникальный ключ (channel_id, message_id). Это позволило в дальнейшем использовать Upsert, что критически важно для соблюдения принципа *атомарности*.

## ⚙️ Установка и Запуск

### 1. Подготовка окружения
- Клонируйте репозиторий и создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```
- Заполните `API_ID` и `API_HASH` (получить на https://my.telegram.org).
- Укажите список каналов для парсинга в `CHANNELS` (в `.env`) или в `channels.txt`.
- Параметры работы парсера берутся из `.env`: `POSTS_LIMIT`, `CONCURRENCY`, `SESSION_NAME`, `LOG_LEVEL`, `NETWORK_RETRIES`, `NETWORK_RETRY_BASE_DELAY_S`.

### 2. Первый запуск (Авторизация)
Поскольку Telethon требует авторизации при первом запуске (ввод номера и кода из SMS), первый запуск контейнера нужно выполнить в интерактивном режиме:

```bash
docker-compose run --rm parser python parser.py
```
> **Важно:** Введите свой номер телефона и код из Telegram. После успешной авторизации в корне проекта появится файл `telethon.session`.

### 3. Штатный запуск сервисов
После создания сессии проект можно запускать одной командой в фоновом режиме:
```bash
docker-compose up -d
```
Эта команда поднимет базу данных PostgreSQL и запустит процесс сбора данных.
